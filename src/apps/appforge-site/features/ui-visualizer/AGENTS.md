# AGENTS.md — `src/apps/appforge-site/features/ui-visualizer`

## Scope

Applies to `src/apps/appforge-site/features/ui-visualizer/**`.

## Purpose

The UI visualizer is a design-time inspector that renders each app screen in a phone-frame canvas and lets users click nodes, inspect token values, and see prop changes reflected live. It is the primary AppForge product surface.

Architecture doc: `docs/architecture/ui-visualizer.md`

---

## Rendering paths

There is **one** rendering path: every screen goes through a LiveLayout component.

- `renderers/live-layout-registry.tsx` maps each document ID to its `*.layout.tsx` component.
- `views/ui-canvas.view.tsx` renders the LiveLayout inside a `VisualizerProvider`.
- `renderers/render-ui-node.tsx` exists for block/fixture previews only — do not use it for screens.

When adding a new screen, register it in `live-layout-registry.tsx`. Do not add a `renderUiNode` branch.

---

## UiDocument and layout files must stay in sync

The UiDocument (in `src/generated/ui-documents.<appId>.ts`) is generated by the scanner. It is the source of truth for the inspector panel; the layout file is the source of truth for the canvas render.

After any change to a `*.layout.tsx` file, re-run:

```
node scripts/scan-ui-documents.mjs example-app
```

Do not edit the generated file by hand.

---

## Selection — two mechanisms, same IDs

Canvas click-to-select uses `data-uiid` DOM attributes. Inspector/layers panel uses `UiDocument` node IDs. Both use the same IDs, so they stay in sync.

- Layout files stamp `ui("nodeId", <Element />)` on every meaningful JSX element.
- IDs must exactly match the node IDs in the scanned UiDocument.
- After changing a layout file's structure (adding/removing/reordering elements), re-run the scanner and re-stamp with the new IDs.

The `ui()` function is in `src/ui/viz.ts`. It calls `React.cloneElement` to attach `__uiid` (for wrapped components) and `data-uiid` (for DOM selection) without touching the component's own props.

---

## Inspector edits semantic props, not raw styles

The inspector shows pickers only for variant contracts:

- Text: `tone`, `size`, `weight`
- Button: `variant`, `size`, `label`
- Container: `bg`, `borderColor`, `borderWidth`, `br`, `gap`, `ai`, `jc`, `p`, `px`, `py`

Layout files must use these semantic props so the scanner captures them and the inspector fields are pre-populated. Do not use raw Tamagui props like `color`, `fontFamily`, or `fontSize` where a semantic contract exists.

---

## Prop overrides — inspector → live canvas flow

```
Inspector change
  → updateSelectedNodeProp(key, value)         [use-ui-playground.ts]
  → applyDocument(updateNodeProps(...))         keeps inspector in sync
  → setLivePropOverrides({ [nodeId]: {key} })   for LiveLayout screens
  → pg.livePropOverrides
  → <UiCanvasView propOverrides={...} />
  → <VisualizerProvider propOverrides={...} />
  → ctx.propOverrides[__uiid]                  read by wrapped barrel components
  → merged into the rendered component's props
```

---

## State

All visualizer state lives in `state/use-ui-playground.ts`. Do not add state to views or renderers.

- `selectedDocumentId` — which screen is open
- `selectedNodeId` — which node is selected (synced by both canvas click and layers panel)
- `docOverrides` — UiDocument edits keyed by `documentId:previewState`
- `livePropOverrides` — per-node prop overrides for LiveLayout screens, cleared on screen switch

## Files and their single responsibilities

| File | Owns |
|---|---|
| `domain/ui-document.types.ts` | Types only: `UiDocument`, `UiNode`, `UiNodeProps` |
| `domain/ui-document.operations.ts` | Pure document transforms |
| `domain/ui-prop-options.ts` | Picker option lists |
| `renderers/live-layout-registry.tsx` | `documentId → React component` map |
| `renderers/render-ui-node.tsx` | Block/fixture fallback renderer |
| `renderers/use-live-node-selection.ts` | DOM click → `data-uiid` → `onSelectNode` |
| `state/use-ui-playground.ts` | All state and derived values |
| `views/ui-canvas.view.tsx` | Phone frame, VisualizerProvider, CSS injection |
| `views/ui-inspector.view.tsx` | Right-panel pickers and layers tree |

---

## Do not

- Use `renderUiNode` for screens.
- Put state in views or renderers.
- Mix raw Tamagui props (`color`, `fontFamily`) with semantic contracts in layout files.
- Edit `src/generated/ui-documents.*.ts` by hand.
- Add `__uiid` or `data-uiid` manually without using `ui()`.
- Introduce a separate selection mechanism other than `data-uiid` DOM walk.
